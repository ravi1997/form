# API Overview

The API is served under `/api/v1` and uses OpenAPI 3 schemas generated by `flask-openapi3`.

## Route groups

### System

- `GET /api/v1/health`
- `GET /api/v1/liveness`
- `GET /api/v1/readiness`
- `GET /api/v1/ready`
- `GET /api/v1/metrics`
- `POST /api/v1/schemas/echo-form`

### Auth

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/change-password`
- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/logout`
- `GET /api/v1/auth/me`
- `GET /api/v1/auth/sessions`
- `POST /api/v1/auth/sessions/revoke`
- `POST /api/v1/auth/logout-all`
- Admin routes under `/api/v1/auth/admin/users/<user_uuid>/...`
- Admin bulk password-change flagging under `/api/v1/auth/admin/users/bulk/must-change-password`

### Organizations

- `/api/v1/organizations`
- `/api/v1/organizations/<uuid>/admins`
- `/api/v1/organizations/<uuid>/invitations`
- `/api/v1/invitations/<uuid>/accept`

### Resources

- `/api/v1/projects`
- `/api/v1/projects/<project_uuid>/forms`
- nested sections, questions, choices, actions, versions, workflow, responses, and effective UI config routes
- `/api/v1/public/projects/<project_uuid>/forms/<form_uuid>/responses`
- action execution history is exposed under the form response tree

### Conditions

- `/api/v1/conditions`
- metadata, testing, cache, monitoring, presets, approval, versioning, bulk, and async routes are implemented under that prefix

### Rate limits

- `/api/v1/admin/rate-limits`

### UI templates

- `/api/v1/ui/theme-templates`
- `/api/v1/ui/layout-templates`

## Authentication conventions

- Protected endpoints use `Authorization: Bearer <access token>`
- Authentication and admin routes emit audit/security events
- Resource routes additionally require RBAC permission checks and generic rate limiting

## Compatibility

Legacy routes under `/api/...` redirect to the `/api/v1/...` equivalents where compatibility is still maintained.
